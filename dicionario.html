📖 Dicionário do Processador

Termos técnicos explicados de forma simples e direta. Use a busca para encontrar rapidamente.

A
ALU Arithmetic Logic Unit
Unidade Lógica e Aritmética da CPU. Executa operações matemáticas (soma, subtração) e lógicas (AND, OR, XOR, comparações). É o "cérebro matemático" do processador.
ASLR Address Space Layout Randomization
Técnica de segurança que randomiza os endereços de memória de stack, heap e bibliotecas a cada execução de um programa. Dificulta ataques que dependem de endereços conhecidos.
Assembly
Linguagem de programação de baixo nível que usa mnemônicos para representar instruções de máquina. Cada instrução Assembly corresponde a uma ou mais instruções de código de máquina. É específica da arquitetura (x86, ARM, MIPS).
B
Barramento Bus
Caminho de comunicação dentro do computador. Existem barramentos de dados (transportam dados), de endereços (indicam posições de memória) e de controle (sinais de comando). Determinam a largura e velocidade de transferência.
Buffer Overflow
Vulnerabilidade onde um programa escreve mais dados em um buffer (área de memória temporária) do que ele pode conter, sobrescrevendo áreas adjacentes da memória. Pode ser explorado para executar código malicioso ou causar crashes.
Byte
Unidade básica de informação digital composta por 8 bits. Pode representar 256 valores diferentes (0-255). É a unidade padrão para medir tamanho de arquivos e memória.
C
Cache
Memória ultra-rápida próxima à CPU que armazena cópias dos dados mais frequentemente usados da RAM. Existem três níveis: L1 (mais rápido, menor), L2 e L3 (maior, mais lento). Reduz drasticamente o tempo de acesso à memória.
Clock
Sinal elétrico que oscila entre ligado e desligado, sincronizando todas as operações da CPU. Medido em Hertz (Hz), com processadores modernos operando em Gigahertz (GHz) — bilhões de ciclos por segundo.
Compilador Compiler
Programa que traduz código fonte de linguagem de alto nível (como C) para código de máquina executável pela CPU. Inclui etapas de pré-processamento, compilação, assembly e linking.
Context Switch
Processo de salvar o estado (registradores, contador de programa) de um processo e carregar o estado de outro. Permite que o sistema operacional execute múltiplos processos aparentemente simultaneamente.
CPU Central Processing Unit
Unidade Central de Processamento. O "cérebro" do computador que executa instruções, processa dados e coordena todas as operações do sistema. Contém a ALU, unidade de controle, registradores e caches.
CU Control Unit
Unidade de Controle da CPU. Coordena todas as operações do processador: busca instruções, decodifica, sinaliza outros componentes e controla o fluxo de execução. É o "maestro" da CPU.
D
DEP Data Execution Prevention
Técnica de segurança que marca regiões de dados (stack, heap) como não executáveis. Impede que código malicioso injetado nessas áreas seja executado pela CPU.
E
Endianness
Ordem em que bytes são armazenados na memória. Little-endian: byte menos significativo primeiro (x86, ARM). Big-endian: byte mais significativo primeiro (network protocols, alguns processadores antigos).
F
Fetch-Decode-Execute
Ciclo básico de execução da CPU. Fetch: busca a próxima instrução na memória. Decode: interpreta o que a instrução faz. Execute: realiza a operação. Repetido bilhões de vezes por segundo.
FPU Floating Point Unit
Unidade de Ponto Flutuante. Componente da CPU especializado em cálculos com números decimais (ponto flutuante). Essencial para gráficos 3D, cálculos científicos e machine learning.
G
GHz Gigahertz
Unidade de frequência igual a 1 bilhão de Hertz (ciclos por segundo). Frequência do clock da CPU. Um processador de 3 GHz executa 3 bilhões de ciclos por segundo.
H
Heap
Região da memória RAM usada para alocação dinâmica. Gerenciada manualmente pelo programador (malloc, calloc, realloc, free). Maior que a stack, mas mais lenta e sujeita a memory leaks.
Hyper-Threading
Tecnologia da Intel onde um núcleo físico se apresenta como dois núcleos lógicos ao sistema operacional. As duas threads compartilham recursos (ALU, FPU) mas têm registradores separados. Ganho de performance: ~15-30%.
I
IDT Interrupt Descriptor Table
Tabela que mapeia cada tipo de interrupção para o endereço da rotina de tratamento (handler) correspondente. Usada pela CPU para saber para onde saltar quando uma interrupção ocorre.
Interrupção Interrupt
Sinal que faz a CPU pausar o programa atual para atender a um evento urgente. Fontes: teclado, mouse, disco, rede, timer. Essencial para multitarefa e resposta a eventos de hardware.
IPC Instructions Per Clock
Número de instruções que a CPU executa por ciclo de clock. CPUs modernas com pipeline e execução fora de ordem podem executar múltiplas instruções por ciclo. Maior IPC = mais performance por GHz.
K
Kernel
Núcleo do sistema operacional. Executa em Kernel Mode (Ring 0) com acesso total ao hardware. Gerencia memória, processos, dispositivos, sistemas de arquivos e fornece serviços via syscalls.
Kernel Mode
Modo de execução privilegiado da CPU onde o sistema operacional roda. Tem acesso total à memória, pode acessar hardware diretamente e modificar tabelas de proteção. Crash no kernel = crash do sistema.
L
Localidade de Referência
Princípio que programas tendem a acessar dados próximos ou recentemente usados. Localidade temporal: reutilizar dados recentes. Localidade espacial: acessar dados próximos na memória. Fundamental para eficiência do cache.
M
Memory Leak
Vazamento de memória. Ocorre quando memória alocada dinamicamente (malloc) nunca é liberada (free). O programa consome cada vez mais memória até esgotar recursos do sistema.
MMU Memory Management Unit
Unidade de Gerenciamento de Memória. Hardware dentro da CPU (ou próximo a ela) que traduz endereços virtuais em endereços físicos. Permite isolamento de processos e proteção de memória.
Multicore
Arquitetura de processador com múltiplos núcleos (cores) independentes no mesmo chip. Cada núcleo é uma CPU completa. Permite verdadeiro paralelismo e melhor eficiência energética que aumentar GHz.
N
Núcleo Core
Unidade de processamento independente dentro de um chip de CPU. Contém ALU, FPU, registradores e caches próprios. Processadores modernos têm de 2 a 64+ núcleos.
O
Opcode
Código de operação. Parte da instrução de máquina que especifica QUAL operação deve ser executada (soma, subtração, movimento, salto, etc.). Cada opcode corresponde a uma ação específica da CPU.
P
Pipeline
Técnica onde a CPU processa múltiplas instruções simultaneamente em diferentes estágios (Fetch, Decode, Execute, Memory, Writeback). Aumenta o throughput sem aumentar a frequência do clock.
Processo Process
Programa em execução com seu próprio espaço de memória, PID (Process ID) e recursos. Isolado de outros processos pelo sistema operacional. Pode conter múltiplas threads.
R
RAM Random Access Memory
Memória de Acesso Aleatório. Memória volátil (perde dados ao desligar) usada como "mesa de trabalho" da CPU. Mais lenta que cache e registradores, mas muito maior (GBs vs KBs).
Registrador Register
Memória ultra-rápida dentro da CPU. Armazena dados temporários para processamento imediato. Em x86_64: RAX, RBX, RCX, RDX (gerais), RIP, RSP, RBP (especiais), RFLAGS (status). Acesso em ~1 ciclo.
RIP Instruction Pointer
Registrador que aponta para o endereço da PRÓXIMA instrução a ser executada. Controla o fluxo do programa. Não pode ser acessado diretamente pelo programador em x86_64.
ROP Return-Oriented Programming
Técnica de exploração que reutiliza pequenos trechos de código existentes (gadgets) em vez de injetar novo código. Usada para burlar proteções como DEP/NX. Defesa: CFI (Control-Flow Integrity).
RSP Stack Pointer
Registrador que aponta para o TOPO da pilha (stack). Diminui em PUSH e aumenta em POP. Crítico para chamadas de função e gerenciamento de variáveis locais.
S
Stack Canary
Valor aleatório colocado entre o buffer e o endereço de retorno na stack. Se um buffer overflow tentar sobrescrever o endereço de retorno, o canary será alterado primeiro. O programa detecta isso e aborta antes de retornar.
Stack Pilha
Região da memória RAM organizada como LIFO (Last In, First Out). Usada para chamadas de função, variáveis locais e endereços de retorno. Gerenciada automaticamente pelo compilador. Cresce para endereços menores.
Syscall System Call
Chamada de sistema. Instrução especial que faz a CPU trocar de User Mode para Kernel Mode para executar operações privilegiadas (ler arquivo, alocar memória, criar processo). Interface entre programas e o sistema operacional.
T
Thread
Sequência de instruções que a CPU executa. Dentro de um processo, múltiplas threads compartilham o mesmo espaço de memória mas têm seus próprios registradores e pilha. Permite concorrência dentro de um programa.
Transistor
Componente eletrônico fundamental que funciona como um interruptor controlado eletricamente. CPUs modernas contêm bilhões de transistores microscopicos. Cada transistor pode estar ligado (1) ou desligado (0).
U
User Mode
Modo de execução não-privilegiado da CPU onde programas de usuário rodam. Acesso limitado à memória, sem acesso direto ao hardware. Se um programa crashar, o sistema continua funcionando. Seguro e isolado.
V
Virtualização
Técnica que permite executar múltiplos sistemas operacionais simultaneamente em um único hardware físico. Usa extensões da CPU (Intel VT-x, AMD-V) para isolar e gerenciar máquinas virtuais de forma eficiente.
X
x86 / x86_64
Arquitetura de processadores da Intel/AMD. x86 = 32-bit (registradores de 32 bits, endereços de 32 bits). x86_64 = 64-bit (registradores de 64 bits, endereços de 64 bits, mais registradores disponíveis). Dominante em PCs e servidores.
