Pilha (Stack)

🩷 Pilha (Stack) — parte essencial da jornada para compreender o funcionamento interno do processador.

📚 O Que é Stack (Pilha)?

A Stack é uma região da memória RAM organizada como uma pilha de pratos: o último a entrar é o primeiro a sair (LIFO — Last In, First Out). A CPU usa a stack para gerenciar chamadas de função, variáveis locais e endereços de retorno.

💡 Por que LIFO?

Quando uma função chama outra, a função chamada DEVE terminar antes da chamadora continuar. A pilha garante que o retorno aconteça na ordem correta — a última função chamada é a primeira a retornar.

⬆️⬇️ Push e Pop

Duas operações fundamentais manipulam a stack:

  • PUSH: Coloca um valor no topo da pilha. O RSP (Stack Pointer) diminui (a pilha cresce para BAIXO na memória).
  • POP: Remove o valor do topo da pilha. O RSP aumenta.
// Em Assembly x86_64:
push rax      // Guarda o valor de RAX na pilha
              // RSP = RSP - 8 (64-bit)
              // [RSP] = valor de RAX

pop rbx       // Recupera o valor do topo para RBX
              // RBX = [RSP]
              // RSP = RSP + 8

📞 Chamadas de Função e Stack Frame

Quando uma função é chamada, a CPU cria um Stack Frame — uma "caixa" na pilha com tudo que a função precisa:

  • Endereço de retorno: Para onde voltar quando a função terminar
  • Parâmetros: Argumentos passados para a função
  • Variáveis locais: Dados temporários da função
  • RBP salvo: O valor anterior do Base Pointer
// Código C:
void func(int a, int b) {
    int local = a + b;  // Variável local na stack
    printf("%d", local);
}

// Assembly equivalente (simplificado):
func:
    push rbp            // Salva o RBP anterior
    mov rbp, rsp        // RBP = topo atual (novo frame)
    sub rsp, 16         // Reserva espaço para variáveis locais

    // a está em [rbp+16], b em [rbp+24]
    // local está em [rbp-4]

    mov eax, [rbp+16]   // eax = a
    add eax, [rbp+24]   // eax += b
    mov [rbp-4], eax    // local = eax

    // ... chama printf ...

    mov rsp, rbp        // Restaura o topo da pilha
    pop rbp             // Restaura o RBP anterior
    ret                 // Retorna (usa endereço salvo)

⚠️ Stack Overflow

A stack tem tamanho LIMITADO (geralmente 8 MB em Linux). Se uma função chamar a si mesma infinitamente ou alocar muitas variáveis locais, a stack "transborda":

// CUIDADO! Stack Overflow:
void recursao_infinita() {
    int buffer[1000];        // Aloca na stack
    recursao_infinita();     // Chama a si mesma → stack cresce!
}

// Cada chamada adiciona um novo frame na pilha
// Eventualmente: STACK OVERFLOW! 💥

// Solução: use o heap para grandes alocações
void correto() {
    int *buffer = malloc(1000 * sizeof(int));  // Heap!
    // ... usa buffer ...
    free(buffer);
}
⚠️ Boas Práticas

Nunca aloque arrays grandes na stack! Use malloc (heap) para buffers maiores que alguns KB. A stack é para dados pequenos e temporários.
