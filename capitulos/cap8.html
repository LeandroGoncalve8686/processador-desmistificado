Heap

🟣 Heap — parte essencial da jornada para compreender o funcionamento interno do processador.

🏔️ O Que é Heap?

O Heap é uma região da memória RAM usada para alocação dinâmica — quando você não sabe de antemão quanto espaço precisa, ou quando precisa de dados que persistem além do escopo de uma função. Diferente da stack, o heap é gerenciado MANUALMENTE pelo programador.

🆚 Stack vs Heap
📚 STACK
✓ Automático (compilador gerencia)
✓ Muito rápido
✗ Tamanho limitado (~8MB)
✗ Dados temporários
✗ Escopo de função
🏔️ HEAP
✓ Tamanho grande (GBs)
✓ Dados persistentes
✓ Controle total do programador
✗ Mais lento (overhead)
✗ Risco de memory leak
💡 Analogia

A Stack é como a mesa de trabalho do escritório — organizada, limitada, tudo limpo no fim do dia. O Heap é como um depósito — enorme, você guarda o que quiser, mas precisa lembrar de organizar e limpar sozinho!

🔧 malloc(), calloc(), realloc(), free()

Em C, quatro funções controlam o heap:

// malloc — aloca memória NÃO inicializada
int *arr = malloc(100 * sizeof(int));
// Retorna um ponteiro para 400 bytes (100 ints)
// Os valores são LIXO (não inicializados!)

// calloc — aloca e ZERA a memória
int *arr2 = calloc(100, sizeof(int));
// Mesmo que malloc, mas todos os bytes = 0

// realloc — redimensiona uma alocação existente
arr = realloc(arr, 200 * sizeof(int));
// Aumenta de 100 para 200 ints
// Pode mover os dados para outro lugar!

// free — LIBERA a memória
free(arr);
// Devolve a memória para o sistema
// APÓS free, NÃO use o ponteiro!
⚠️ Erros Comuns
  • Memory Leak: Esquecer de chamar free() → memória nunca é liberada
  • Use After Free: Usar ponteiro depois de free() → comportamento indefinido
  • Double Free: Chamar free() duas vezes no mesmo ponteiro → crash
  • NULL pointer: malloc pode retornar NULL se não houver memória

🧠 Como o Processador Acessa o Heap

Quando você chama malloc, o sistema operacional encontra um bloco livre no heap e retorna seu endereço. A CPU acessa esse endereço como qualquer outro:

// C:
int *ptr = malloc(sizeof(int) * 5);
*ptr = 10;           // Escreve no heap
*(ptr + 1) = 20;     // Escreve no próximo int
ptr[2] = 30;         // Mesma coisa, notação array

// Assembly equivalente:
// call malloc           // SO aloca memória
// mov [rax], 10        // *ptr = 10 (RAX = endereço retornado)
// mov [rax+4], 20      // ptr[1] = 20 (int = 4 bytes)
// mov [rax+8], 30      // ptr[2] = 30

// O endereço no RAX é arbitrário — depende do SO
// A CPU não sabe se é stack, heap ou .data!
// Ela apenas lê/ESCREVE no endereço dado
🗺️ Layout de Memória de um Programa
Stack (cresce ↓) — Variáveis locais, frames
↓ Espaço livre ↓
Heap (cresce ↑) — malloc, calloc, realloc
.bss — Variáveis não inicializadas
.data — Variáveis globais/static
.text — Código do programa (read-only)
