Núcleos e Threads

🟡 Núcleos e Threads — parte essencial da jornada para compreender o funcionamento interno do processador.

🧬 O Que é um Núcleo (Core)?

Um núcleo é uma CPU completa e independente dentro do mesmo chip físico. Antigamente, processadores tinham apenas 1 núcleo. Hoje, é comum ter 4, 8, 16 ou até mais núcleos no mesmo processador.

🏗️ Evolução dos Processadores
🕰️ Single-Core (2000s)
1 núcleo, 1 thread. Tudo sequencial. GHz cada vez maiores, mas limitação física do calor.
⚡ Dual-Core (2005+)
2 núcleos independentes. Dois programas podem rodar verdadeiramente ao mesmo tempo.
🚀 Quad-Core / Octa-Core (2010+)
4-8 núcleos. Padrão em desktops e notebooks modernos.
🔥 Many-Core (2020+)
16, 32, 64+ núcleos. Servidores, workstations, CPUs de alta performance.
💡 Por que múltiplos núcleos?

Aumentar a frequência (GHz) de um único núcleo gera muito calor e consome muita energia. É mais eficiente ter vários núcleos trabalhando juntos. É como ter vários cozinheiros em uma cozinha em vez de um super-cozinheiro!

🧵 Threads e Hyper-Threading

Uma thread é uma sequência de instruções que a CPU executa. Cada programa tem pelo menos uma thread principal.

Hyper-Threading (Intel) ou SMT (Simultaneous Multithreading, AMD) é uma tecnologia onde UM núcleo físico se apresenta como DOIS núcleos lógicos ao sistema operacional:

🔀 Hyper-Threading: 1 Núcleo Físico = 2 Lógicos
🏗️ Núcleo Físico
1 ALU, 1 FPU, 1 Cache L1 (compartilhado)
↓
🧵 Thread 1
Registradores + PC
🧵 Thread 2
Registradores + PC
  • As duas threads COMPARTILHAM a ALU, FPU e caches
  • Quando uma thread espera (cache miss, por exemplo), a outra usa o núcleo
  • Ganho de performance: ~15-30% (nunca 100% como dois núcleos reais)
🤔 Curiosidade

Em Linux, você pode ver os núcleos lógicos com lscpu ou nproc. Se mostrar 8 mas você tem um processador quad-core, significa que tem Hyper-Threading ativo (4 físicos × 2 = 8 lógicos).

⚡ Paralelismo vs Concorrência

Dois conceitos importantes e frequentemente confundidos:

🆚 Paralelismo vs Concorrência
⚡ Paralelismo
✓ Múltiplas tarefas executam SIMULTANEAMENTE
✓ Requer múltiplos núcleos
✓ Exemplo: 2 vídeos sendo renderizados em 2 núcleos
✓ "Fazendo várias coisas AO MESMO TEMPO"
🔄 Concorrência
✓ Múltiplas tarefas progridem no mesmo período
✓ Pode acontecer em 1 núcleo (alternando)
✓ Exemplo: SO alternando entre apps rapidamente
✓ "Gerenciando várias coisas ao mesmo tempo" 
// Em C, você pode criar threads com pthreads:
#include 

void* tarefa(void* arg) {
    printf("Executando em thread!
");
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, tarefa, NULL);  // Cria thread 1
    pthread_create(&t2, NULL, tarefa, NULL);  // Cria thread 2

    pthread_join(t1, NULL);  // Espera thread 1 terminar
    pthread_join(t2, NULL);  // Espera thread 2 terminar
    return 0;
}

// Cada thread pode rodar em um núcleo diferente!
// Se houver 2 núcleos, as threads rodam em PARALELO
// Se houver 1 núcleo, as threads rodam em CONCORRÊNCIA

🧩 Mapa Mental — Núcleos e Threads
🧬 Núcleo
├─ CPU independente
├─ ALU + FPU + Cache
├─ 1 núcleo = 1 unidade
└─ Multicore = vários
🧵 Thread
├─ Sequência de instruções
├─ 1 thread por padrão
├─ Multithreading = várias
└─ Compartilha recursos
🔀 Hyper-Threading
├─ 1 físico = 2 lógicos
├─ Compartilha ALU/FPU
├─ ~15-30% ganho
└─ Não é 2 núcleos reais
