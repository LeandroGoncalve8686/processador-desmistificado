Interrupções

🟢 Interrupções — parte essencial da jornada para compreender o funcionamento interno do processador.

🔔 O Que é uma Interrupção?

Uma interrupção é um sinal que faz a CPU PAUSAR o que está fazendo para atender a algo mais urgente. É como um telefone tocando enquanto você cozinha — você para o fogão, atende o telefone, e depois volta a cozinhar.

📞 Analogia da Interrupção
🍳 CPU executando programa normalmente
↓
🔔 INTERRUPÇÃO! (teclado, timer, disco...)
↓
💾 Salva estado atual (registradores) na stack
↓
⚡ Executa rotina de tratamento da interrupção
↓
🔄 Restaura estado e volta ao programa
💡 Por que interrupções existem?

Sem interrupções, a CPU teria que ficar verificando constantemente se o teclado foi pressionado, se o disco terminou de ler, se a rede recebeu pacotes... Isso seria um desperdício enorme. Com interrupções, a CPU trabalha normalmente e SÓ reage quando algo acontece.

🖱️ Tipos de Interrupções

Interrupções podem vir de várias fontes:

🔌 Fontes de Interrupção
⌨️ Teclado
Quando você pressiona uma tecla, o controlador do teclado envia uma interrupção para a CPU processar o caractere.
🖱️ Mouse
Movimentos e cliques geram interrupções constantes para atualizar o cursor na tela.
💿 Disco (SSD/HD)
Quando uma leitura/escrita termina, o disco interrompe a CPU para entregar os dados.
🌐 Rede
Pacotes de rede chegando geram interrupções para processamento imediato.
⏱️ Timer (Relógio)
Interrompe a CPU milhares de vezes por segundo para permitir multitarefa (escalonamento de processos). Essencial para o funcionamento do sistema operacional!

⚙️ Fluxo Completo de uma Interrupção

Vamos acompanhar o que acontece quando você pressiona uma tecla:

// Passo a passo de uma interrupção de teclado:

1. Você pressiona 'A'
   ↓
2. Controlador do teclado detecta o evento
   ↓
3. Controlador envia sinal de interrupção (IRQ1) para a CPU
   ↓
4. CPU PAUSA o programa atual
   ↓
5. CPU salva registradores (RAX, RBX, RFLAGS...) na stack
   ↓
6. CPU consulta a IDT (Interrupt Descriptor Table)
   ↓
7. CPU salta para a rotina de tratamento (handler)
   ↓
8. Handler lê o scan code da tecla do controlador
   ↓
9. Handler converte scan code para caractere ASCII
   ↓
10. Handler entrega o caractere ao sistema operacional
   ↓
11. SO coloca o caractere no buffer do terminal
   ↓
12. CPU restaura registradores da stack
   ↓
13. CPU retorna ao programa original (IRET)

// Tudo isso acontece em MICROSSEGUNDOS!
🤔 Curiosidade

Em Linux, você pode ver as interrupções que ocorrem com o comando cat /proc/interrupts. Cada linha mostra quantas vezes cada dispositivo interrompeu cada núcleo da CPU!
