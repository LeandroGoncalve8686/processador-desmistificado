Cache

🔵 Cache — parte essencial da jornada para compreender o funcionamento interno do processador.

⚡ O Que é Cache?

Cache é uma memória ultra-rápida que fica DENTRO ou muito próxima da CPU. Ela armazena cópias dos dados mais frequentemente usados da RAM, para que a CPU não precise esperar toda vez que precisar de algo.

📊 Pirâmide de Memória — Velocidade vs Capacidade
⚡ Registradores
~1 ns | ~1 KB total | Mais rápido
💨 Cache L1
~2-4 ns | 32-64 KB por núcleo
🚀 Cache L2
~10 ns | 256 KB - 1 MB por núcleo
🏎️ Cache L3
~20-40 ns | 8-64 MB (compartilhado)
💾 RAM (Memória Principal)
~100 ns | 8-64 GB | Mais lenta
💡 Por que Cache Existe?

A RAM é muito lenta comparada à CPU. Sem cache, a CPU passaria a maior parte do tempo ESPERANDO. O cache funciona como uma "memória de bolso" com os dados mais usados.

🔍 Cache L1, L2 e L3

CPUs modernas têm múltiplos níveis de cache, cada um com características diferentes:

🏗️ Hierarquia de Cache em um CPU Moderno
⚡ L1 Cache
• 32-64 KB por núcleo
• Dividido em L1d (dados) e L1i (instruções)
• ~4 ciclos de latência
• Mais rápido, menor
🚀 L2 Cache
• 256 KB - 1 MB por núcleo
• Instruções + dados juntos
• ~10-12 ciclos de latência
• Meio-termo
🏎️ L3 Cache
• 8-64 MB (compartilhado entre todos os núcleos)
• Última parada antes da RAM
• ~30-40 ciclos de latência
• Maior, mais lento que L1/L2
🤔 Curiosidade

O L1i (cache de instruções) e L1d (cache de dados) são separados porque a CPU busca instruções e dados simultaneamente. Ter caches separados permite buscar ambos ao mesmo tempo — isso é chamado de arquitetura Harvard modificada!

📈 Cache Hit vs Cache Miss

Quando a CPU precisa de um dado, ela verifica o cache primeiro:

  • Cache Hit: O dado está no cache! A CPU pega instantaneamente. 🎉
  • Cache Miss: O dado NÃO está no cache. A CPU precisa buscar na RAM (ou L3 → L2 → L1). ⏳
🎯 Cache Hit vs Miss
✅ Cache Hit
1. CPU precisa de dado
2. Verifica L1 → ENCONTRA!
3. ~4 ciclos
4. Programa continua rápido
❌ Cache Miss
1. CPU precisa de dado
2. Verifica L1 → NÃO está
3. Verifica L2 → NÃO está
4. Verifica L3 → NÃO está
5. Busca na RAM: ~100-300 ciclos!
6. Programa fica lento
⚠️ Impacto no Desempenho

Um cache miss pode custar 100x mais ciclos que um hit! Programas bem otimizados tentam manter dados frequentemente usados próximos uns dos outros na memória (localidade espacial) e reutilizar dados recentes (localidade temporal).

💻 Localidade de Referência

Programas bem escritos aproveitam duas propriedades do cache:

  • Localidade Temporal: Se você usou um dado agora, provavelmente usará de novo em breve. Exemplo: variáveis de loop.
  • Localidade Espacial: Se você usou um dado, provavelmente usará dados próximos a ele. Exemplo: arrays percorridos sequencialmente.
// Código COM boa localidade (rápido):
int arr[1000];
for (int i = 0; i < 1000; i++) {
    arr[i] = i;  // Acessa sequencialmente → Cache adora!
}

// Código COM má localidade (lento):
int arr[1000][1000];
for (int i = 0; i < 1000; i++) {
    for (int j = 0; j < 1000; j++) {
        arr[j][i] = 0;  // Salta na memória → Cache sofre!
    }
}
💡 Relação com C

Em C, entender cache ajuda a escrever código mais rápido. Arrays multidimensionais em C são armazenados em row-major order (linha por linha). Acessar arr[i][j] sequencialmente é rápido, mas arr[j][i] salta na memória e causa muitos cache misses.
