Como um Programa é Executado

🟣 Como um Programa é Executado — parte essencial da jornada para compreender o funcionamento interno do processador.

🔄 O Ciclo Fetch-Decode-Execute

Todo programa que você escreve em C, Python ou JavaScript precisa passar por uma transformação extraordinária antes de chegar à CPU. Vamos acompanhar essa jornada passo a passo.

🛤️ Jornada do Código até a CPU
📝 Código em C
int x = 10; printf("%d", x);
⚙️ Compilador (gcc/clang)
Traduz para linguagem intermediária
🔧 Assembly (x86_64)
mov eax, 10 | call printf
💻 Código de Máquina (Binário)
10110000 00001010 00000000...
🧠 CPU Executa
Fetch → Decode → Execute (repete)

📥 Fetch — Buscando a Instrução

A CPU usa um registrador especial chamado RIP (Instruction Pointer) para saber qual instrução buscar na memória:

  • O RIP contém o ENDEREÇO da próxima instrução na RAM
  • A CPU lê esse endereço e busca a instrução na memória
  • A instrução é copiada para dentro da CPU
  • O RIP é automaticamente incrementado para apontar para a próxima instrução
// Em Assembly x86_64, o RIP aponta para:
0x401000:  mov eax, 10      // RIP aponta aqui → FETCH
0x401005:  add eax, 5       // Próxima instrução
0x40100A:  mov [rbp-4], eax // Próxima instrução
💡 Analogia

O Fetch é como um aluno pegando a próxima página do livro para ler. O RIP é o marcador de página que diz "comece daqui".

🔍 Decode — Decodificando a Instrução

A instrução buscada é apenas uma sequência de bits (0s e 1s). A CPU precisa entender o que fazer:

  • Opcode: Os primeiros bits dizem QUAL operação (soma, subtração, movimento, etc.)
  • Operandos: Os bits seguintes dizem COM QUAIS dados operar (registradores, memória, valores imediatos)
  • A Unidade de Controle (CU) interpreta tudo isso e prepara a ALU/FPU para a ação
🔢 Exemplo de Decodificação
10110000
Opcode: MOV
+
00001010
Operando: 10
=
MOV EAX, 10
"Mova o valor 10 para o registrador EAX"

⚙️ Execute — Executando a Operação

Agora a CPU sabe EXATAMENTE o que fazer:

  • Se for uma operação aritmética → ALU executa
  • Se for um número decimal → FPU executa
  • Se for acessar memória → Barramento de endereços é usado
  • Se for um salto (jump) → RIP é alterado para um novo endereço
// Exemplo completo em C e Assembly:

// Código C:
int main() {
    int a = 10;      // mov DWORD PTR [rbp-4], 10
    int b = 20;      // mov DWORD PTR [rbp-8], 20
    int c = a + b;   // mov eax, [rbp-4]
                     // add eax, [rbp-8]
                     // mov [rbp-12], eax
    return 0;        // mov eax, 0
}
🤔 Curiosidade

Um processador moderno de 3 GHz pode executar o ciclo Fetch-Decode-Execute 3 bilhões de vezes por segundo em CADA núcleo! Um programa simples como o acima leva nanossegundos para executar.

📐 Pipeline — Execução Paralela Dentro da CPU

CPUs modernas não esperam uma instrução terminar para começar a próxima. Elas usam pipeline:

🚰 Pipeline de 5 Estágios
IF
Fetch
ID
Decode
EX
Execute
MEM
Memory
WB
Write Back
Ciclo 1: IF(1) | - | - | - | -
Ciclo 2: IF(2) | ID(1) | - | - | -
Ciclo 3: IF(3) | ID(2) | EX(1) | - | -
Ciclo 4: IF(4) | ID(3) | EX(2) | MEM(1) | -
Ciclo 5: IF(5) | ID(4) | EX(3) | MEM(2) | WB(1)
→ 5 instruções em andamento simultaneamente!
⚠️ Pipeline Hazards

Quando uma instrução depende do resultado da anterior, o pipeline precisa esperar. Isso é chamado de data hazard e reduz a performance. Compiladores otimizam o código para minimizar esses casos.
