Memória e Processador

🟠 Memória e Processador — parte essencial da jornada para compreender o funcionamento interno do processador.

🔗 O Relacionamento entre CPU e RAM

A CPU e a RAM são como um cérebro e sua mesa de trabalho. A CPU precisa constantemente buscar dados da RAM para processar, e depois devolver os resultados. Essa comunicação é fundamental para entender como programas funcionam.

🔄 Comunicação CPU ↔ RAM

→
🚌 Barramento
Endereço + Controle
→
💾 RAM

💡 Analogia

A CPU é o cozinheiro, a RAM é a geladeira. O cozinheiro pega ingredientes (dados) da geladeira, prepara na bancada (registradores), e guarda o pronto de volta. Se a geladeira estiver longe (RAM lenta), o cozinheiro fica esperando!

📍 Endereços de Memória

Cada byte na RAM tem um endereço único, como o número de cada casa em uma rua. Em sistemas 64-bit, esses endereços podem ir de 0x0000000000000000 até 0xFFFFFFFFFFFFFFFF — teoricamente 16 exabytes de memória endereçável!

// Em C, cada variável tem um endereço na memória:
int x = 10;           // Suponha que x está no endereço 0x7FFF1234

// O operador & retorna o endereço:
printf("%p", &x);     // Imprime: 0x7FFF1234

// O valor 10 está ARMAZENADO nesse endereço
// A CPU acessa assim: mov eax, [0x7FFF1234]

🎯 Variáveis na Memória

Quando você declara uma variável em C, o compilador decide onde colocá-la na memória:

  • Variáveis locais: Vão para a pilha (stack) — rápido, temporário
  • Variáveis globais/estáticas: Vão para a seção de dados — permanente durante a execução
  • Variáveis dinâmicas (malloc): Vão para o heap — controle manual do programador
// Exemplo completo de alocação de memória:

int global = 100;           // Seção .data (endereço fixo)

void func() {
    int local = 50;         // Stack (endereço relativo a RBP)
    static int estatica = 1; // Seção .data
    int *heap = malloc(4);  // Heap (endereço retornado pelo SO)
    *heap = 200;

    // A CPU acessa cada um de forma diferente!
    // global  → mov eax, [endereco_fixo]
    // local   → mov eax, [rbp-4]
    // *heap   → mov eax, [endereco_dinamico]
}

🔗 Ponteiros em C e a CPU

Ponteiros são a ponte entre programação de alto nível e hardware. Um ponteiro é simplesmente um endereço de memória:

// Em C:
int x = 10;
int *ptr = &x;     // ptr contém o ENDEREÇO de x

// Na CPU (Assembly x86_64):
// mov DWORD PTR [rbp-4], 10     // x = 10 (na stack)
// lea rax, [rbp-4]              // rax = &x (endereço de x)
// mov QWORD PTR [rbp-16], rax   // ptr = rax (guarda endereço)
// mov rax, [rbp-16]             // rax = ptr
// mov eax, [rax]                // eax = *ptr = valor em x = 10

// A instrução LEA (Load Effective Address) calcula endereços
// A instrução MOV com [] acessa o VALOR no endereço
🔍 Como a CPU Vê Ponteiros
📦 Variável x
Endereço: 0x7FFF1234 | Valor: 10
↑ &x (endereço)
🔗 Ponteiro ptr
Endereço: 0x7FFF1228 | Valor: 0x7FFF1234 (aponta para x)
🤔 Curiosidade

Em arquitetura x86_64, ponteiros ocupam 8 bytes (64 bits). Em x86 (32-bit), ocupam 4 bytes. Por isso programas 64-bit usam mais memória — mas podem endereçar muito mais!
